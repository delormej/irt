00742  * the target setting level.  By default, Xcode will add a path to a
00743  * specific build when adding the API library to the target.  This target

00777  * <h1>Wahoo Fitness API v2.0 Release Notes</h1>
00778  * <h2>BREAKING CHANGES</h2>
00779  * Up until this release, we have endeavored to keep breaking changes to a
00780  * minimum.  However, the additions to this release required a major revision of
00781  * the architecture.  We apologize for the inconvenience.  We hope the
00782  * improvements will be worth the trouble of upgrading.
00783  *
00784  * <h3>Overview of Changes</h3>
00785  * <ul>
00786  * <li>The WFHardwareConnector singleton is now enforced.  It is available through
00787  * the WFHardwareConnector::sharedConnector method.</li>
00788  *
00789  * <li>All sensor connections are now managed through the WFSensorConnection
00790  * object.  An instance is obtained via the WFHardwareConnector::requestSensorConnection:
00791  * method.</li>
00792  *
00793  * <li>WFSensorConnection and WFConnectionParams facilitate sensor pairing and
00794  * connecting to previously paired sensors.  Up to four previously paird sensors
00795  * may be specified in the connection request.  The API will connect to the
00796  * first available of the specified devices.
00797  *
00798  * <li>WFSensorConnection derived classes now provide access to sensor-specific
00799  * commands.  For example, the WFBikePowerConnection::setBikePowerCalibration:
00800  * method provides a way for the application to set the calibration on the
00801  * Bike Power sensor.</li>
00802  *
00803  * <li>Sensor data is now provided in the WFSensorData derived classes.
00804  * Instances are obtained through the WFSensorConnection::getData method.</li>
00805  * </ul>

00890  * <h2>IPHONE OS 4.0</h2>
00891  * <h3>Background Mode</h3>
00892  * Beginning in iOS 4.0, applications can be run in the background.  This
00893  * is not entirely true.  Background operations are limited to a few services
00894  * provided by the iOS.  Otherwise, the application is suspended.  It is
00895  * kept in memory, but no processing is performed.  At this time, Apple does
00896  * not allow an attached accessory (such as the fisica key) to continue
00897  * communication with the application.  In the previous release notes, it was
00898  * indicated that the Wahoo Fitness API had been tested and confirmed working
00899  * in the following cases:
00900  * <br />
00901  * <ul>
00902  * <li>Top-lock button is pressed.</li>
00903  * <li>User answers an incoming phone call.</li>
00904  * <li>Incoming SMS pop-up message is dismissed (without replying).</li>
00905  * <li>User accesses iPod controls (without opening full iPod app).</li>
00906  * </ul>
00907  * It turns out that this was incorrect.  More detailed tests determined what
00908  * occurs when the top-lock is pressed or a phone call is answered.  When the
00909  * incoming call screen or incoming SMS popup is displayed, the application
00910  * receives the applicationWillResignActive message.  This indicates that
00911  * the application may be about to enter background mode.  If the user
00912  * answers the phone call, or chooses to view the SMS, or presses the top-lock
00913  * button, the application will receive the applicationDidEnterBackground after
00914  * 15 seconds.  The bottom line is that if the application is resumed within 15
00915  * seconds, there is no background mode, and no interruption of normal operation
00916  * of the API.  After 15 seconds, the application enters background and the
00917  * accessory (fisica key) is disconnected.
00918  *
00919  * Although Apple does not allow communication with the accessory to
00920  * continue in background mode at this time, there may be a workaround to
00921  * keep the data flowing.  If a workaround is discovered, or if Apple changes
00922  * the background mode policy, we will post an update.
00923  * <br /><br /><br /><br /><br />

00927  * <h1>Wahoo Fitness API v1.5 Release Notes</h1>
00928  * <h2>BREAKING CHANGES</h2>
00929  * <h3>Method Signature Modifications</h3>
00930  * The signature of the WFHardwareConnector::isSensorConnected:isSearching:
00931  * method was changed in this version to add the status of sensors in "search"
00932  * mode.  This is where a connection request has been initiated, but
00933  * communication has not yet been established.  The previous version of this
00934  * method would return <c>TRUE</c> only when communication to the sensor had been
00935  * established.  Adding <code>isSearching:nil</code> to the previous version
00936  * method call should preform the same as in previous versions, and allow
00937  * existing code to build. To check for a sensor in search mode, pass a pointer
00938  * to a boolean variable.
00939  * <p>
00940  * <h3>Firmware Version</h3>
00941  * The WFHardwareConnector initialization routine in this version requires
00942  * fisica device firmware version 0.9.1 or higher.  Devices with earlier
00943  * firmware version will perform unpredictably, at best.  If your device
00944  * has an earlier version of the firmware, please contact Wahoo Fitness
00945  * for information about upgrading.
00946  * <br /><br /><br />
00947  *
00948  * <h2>BUG FIXES</h2>
00949  * <h3>Connect Sensor Issue</h3>
00950  * Previous version of the API had an issue where calling
00951  * WFHardwareConnector::connectSensorType:withId: would return success when
00952  * the fisica device was not connected.  The enum value
00953  * <c>WF_CONNECT_DEVICE_WRONG_HW_STATE</c> value was added to the
00954  * <c>WFConnectDeviceResult_t</c> enum to handle this situation.
00955  * <h3>Data Notification</h3>
00956  * In previous versions of the API, the
00957  * WFHardwareConnectorDelegate::hardwareConnector:hasData: method was invoked
00958  * at the sample rate regardless of whether new data was avaliable.  This has
00959  * been fixed such that the method is invoked only when new data is available
00960  * for one or more of the connected sensors.
00961  * <br /><br /><br />
00962  *
00963  * <h2>ADDITIONS AND MODIFICATIONS</h2>
00964  * <h3>Deprecated Methods</h3>
00965  * The WFHardwareConnectorDelegate::hardwareConnector:connectionStatus: method
00966  * is deprecated in version 1.5 of the API.  This method will be removed in the
00967  * next release.  Existing code should be modified to use the
00968  * WFHardwareConnectorDelegate::hardwareConnector:stateChanged: method instead.
00969  * This method will provide more information about the current state of the
00970  * WFHardwareConnector.  When implementing this method, checking the
00971  * ::WF_HWCONN_STATE_ACTIVE bit of the
00972  * <i>currentState</i> parameter will provide the same information that the
00973  * <i>connected</i> parameter of the
00974  * WFHardwareConnectorDelegate::hardwareConnector:connectionStatus: method
00975  * provides.  For example:
00976  * <br /><br />
00977  * <code>
00978  * (void)hardwareConnector:(_WFHardwareConnector*)hwConnector connectionStatus:(BOOL)connected
00979  * <br />
00980  * {
00981  * <br />
00982  * &nbsp;&nbsp;BOOL isDeviceConnected = connected;  // this value is the same as...
00983  * <br />
00984  * }
00985  * <br />
00986  * <br />
00987  * (void)hardwareConnector:(_WFHardwareConnector*)hwConnector stateChanged:(WFHardwareConnectorState_t)currentState
00988  * <br />
00989  * {
00990  * <br />
00991  * &nbsp;&nbsp;BOOL isDeviceConnected = (currentState & WF_HWCONN_STATE_ACTIVE);  // ...this value.
00992  * <br />
00993  * }
00994  * </code>
00995  * <p>

00998  * information about the state of the WFHardwareConnector.

01007  * <h2>IPHONE OS 4.0</h2>
01008  * <h3>Supported Platforms</h3>
01009  * Version 1.5 of the Wahoo Fitness API is built against iPhone SDK 4.0.  The
01010  * target device family is iPhone/iPad, and the OS Deployment Target is iPhone
01011  * OS 3.0.  Therefore, the API should build with any application project
01012  * targeting iPhone or iPad with iOS version 3.0 or higher.  It has been tested
01013  * with iPhone devices running iOS 3.1.2, 3.1.3 and 4.0, as well as the iPad.
01014  * There is no internal code or methods on the public interface which use
01015  * features specific to the iOS 4.0 SDK.
01016  * <p>
01017  * <h3>Background Mode</h3>
01018  * The iPhone OS 4.0 introduced the ability for applications to run in the
01019  * background.  Support for background processing is somewhat limited, and the
01020  * resources available are few.  An application which enters background mode
01021  * is typically suspended, and no processing is performed.  A select set of
01022  * services may be requested by the application, which when available and
01023  * granted are allowed to be processed in the background.  The service which
01024  * is most useful to users of the Wahoo Fitness API is likely the Location
01025  * service.  An application may request to have the Location service
01026  * deliver updates while in background mode.
01027  * <br /><br />
01028  * Unfortunately, though receiving Location service updates may keep the
01029  * application from being suspended while in the background, it will not enable
01030  * access to external accessories.  At this time, Apple does not provide access
01031  * to external accessories while in the background.  So, while the API may be
01032  * accessed from an application running in the background, it will behave the
01033  * same as if the fisica key were not inserted.  Therefore, no sensor
01034  * connections or data updates will be available while in the background.
01035  * <br /><br />
01036  * When the application enters background mode, the OS will notify the API that
01037  * the external accessory is not longer available.  The API will then invoke
01038  * the WFHardwareConnectorDelegate::hardwareConnector:stateChanged: method
01039  * to inform the application that the fisica key was disconnected.  
01040  * (The WFHardwareConnectorDelegate::hardwareConnector:connectionStatus: method
01041  * is also invoked, though this method was deprecated in this API version and
01042  * will not be supported in future API releases).  When the application is
01043  * resumed from background, the API will invoke
01044  * the WFHardwareConnectorDelegate::hardwareConnector:stateChanged: method
01045  * again to inform the application that the fisica key is connected (assuming
01046  * that the key is still physically connected to the iPhone).
01047  * <br /><br />
01048  * The Wahoo Fitness API has been tested and confirmed to perform normally
01049  * (connections maintained, and data updated), in several common cases:
01050  * <br />
01051  * <ul>
01052  * <li>Top-lock button is pressed.</li>
01053  * <li>User answers an incoming phone call.</li>
01054  * <li>Incoming SMS pop-up message is dismissed (without replying).</li>
01055  * <li>User accesses iPod controls (without opening full iPod app).</li>
01056  * </ul>
